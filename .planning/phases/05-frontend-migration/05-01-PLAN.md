Purpose: O backend de termos está pronto. O frontend ainda persiste em localStorage — isso deve ser eliminado para que os dados fiquem no PostgreSQL por usuário.
Output: App.tsx sem nenhuma referência a localStorage. Dados lidos/salvos via API REST.
</objective>

Substituir toda a lógica de persistência localStorage por chamadas fetch:

- Carregar lista de termos: `GET /api/terms` (ao montar o componente)

- Deletar termo: `DELETE /api/terms/:id`
- Buscar termo por ID: `GET /api/terms/:id` (se necessário para edição)

Padrão para lidar com estado de loading e erros:
- Mostrar loading enquanto fetch está em andamento
- Mostrar mensagem de erro se fetch falhar (não crashar silenciosamente)
- Manter o comportamento UX existente: após salvar, listar; após deletar, remover da lista

Não alterar nenhuma outra lógica (formulário, preview, abas, etc.) — apenas a camada de persistência.
