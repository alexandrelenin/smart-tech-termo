<task type="auto">
  <name>Task 1: Verificar pré-requisitos do .env</name>
  <files>.env</files>
  <action>
Verificar que o arquivo `.env` existe e contém as variáveis necessárias para o Docker:
- DATABASE_URL
- BETTER_AUTH_SECRET
- BETTER_AUTH_URL
- GEMINI_API_KEY

Se alguma variável estiver faltando, reportar quais estão ausentes. Não abortar — apenas documentar.

**Nota:** GOOGLE_CLIENT_ID e GOOGLE_CLIENT_SECRET são opcionais para o smoke test básico.
  </action>
  <verify>
    <automated>test -f .env && echo ".env exists" || echo "WARNING: .env not found"</automated>
  </verify>
  <acceptance_criteria>
    - .env existe com pelo menos DATABASE_URL e BETTER_AUTH_SECRET
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 2: Build das imagens Docker</name>
  <action>
Rodar o build das imagens para verificar que os Dockerfiles não têm erros:

```bash
docker compose build --no-cache 2>&1
```

Se o build falhar, capturar o erro e reportar qual serviço falhou e por quê.
Se o build passar, registrar: "Build OK — api e nginx compilados com sucesso".
  </action>
  <verify>
    <automated>docker compose images 2>/dev/null | grep -E "(api|nginx)" | wc -l</automated>
  </verify>
  <acceptance_criteria>
    - `docker compose build` termina com exit code 0
    - Imagens api e nginx aparecem em `docker compose images`
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 3: Subir os serviços e smoke test</name>
  <action>
Iniciar os serviços em background e testar os endpoints:

```bash
docker compose up -d 2>&1
sleep 10
# Testar health check do backend
curl -s -o /dev/null -w "%{http_code}" http://localhost:3001/api/health
# Testar frontend via nginx
curl -s -o /dev/null -w "%{http_code}" http://localhost:8095
```

Se os serviços não subirem, verificar logs:
```bash
docker compose logs --tail=30
```

Após o teste, derrubar os serviços:
```bash
docker compose down
```
  </action>
  <verify>
    <automated>docker compose ps --format json 2>/dev/null | head -5 || echo "services not running (expected after down)"</automated>
  </verify>
  <acceptance_criteria>
    - `GET http://localhost:3001/api/health` retorna 200
    - `GET http://localhost:8095` retorna 200
    - `docker compose down` finaliza sem erros
  </acceptance_criteria>
</task>

</tasks>
