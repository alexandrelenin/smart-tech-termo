## Summary

Single-page application (SPA) built with React 19 and Vite. All state lives in a single root component (`App.tsx`) and is persisted to `localStorage`. There is no backend â€” the Gemini API is called directly from the browser with an exposed API key. The planned migration adds a Hono/Node.js backend, PostgreSQL via Prisma, and Better Auth for authentication.

## Current Pattern

**Overall:** Frontend-only SPA, monolithic component with inline persistence.

**Key Characteristics:**
- All business logic, form state, PDF generation, and persistence are co-located in `App.tsx` (396 lines)
- No routing â€” a `showPreview` boolean flag toggles between editor and document preview
- No global state library; `useState` / `useEffect` in `App.tsx` is the entire state layer
- Persistence via `localStorage` under key `smart_tech_reports_db` (array of `ReportData` objects serialized as JSON)
- Gemini API called directly from `services/gemini.ts` in the browser â€” API key exposed via `process.env.API_KEY` resolved at build time by Vite

## Data Flow

**Create / Edit Term:**
1. User fills form fields in `App.tsx`
2. `handleUpdate(path, value)` uses dot-notation path to update nested `ReportData` state via `setData`
3. License items updated via `updateItem(id, field, value)` which maps over `data.items`
4. State lives in `useState<ReportData>` in `App.tsx` â€” no derived/cached state

**Save Term:**
1. User clicks "Salvar no HistĂłrico"
2. `saveToDatabase()` upserts current `data` into `savedReports` array (match by `id`)
3. Full array written to `localStorage.setItem(DB_KEY, JSON.stringify(...))`
4. `setSaveStatus` drives the button's loading/success UI

**Load Term:**
1. On mount, `useEffect` reads `localStorage.getItem(DB_KEY)` and sets `savedReports`
2. User clicks load button in "Banco" tab â†’ `setData(report)` replaces active editor state

**PDF Export:**
1. User navigates to preview mode (`setShowPreview(true)`)
2. `ReportPreview` renders the document in A4 layout
3. "Baixar PDF" calls `html2pdf()` (loaded via CDN script tag in `index.html`) on `reportRef.current`
4. Falls back to `window.print()` on error

**AI Improvement:**
1. `improveDescription(text, type)` sends text to Gemini and returns improved string
2. `generateSummary(items)` generates a formal conclusion paragraph
3. Note: `services/gemini.ts` exists but no call sites are present in the current `App.tsx` â€” UI buttons for this feature may have been removed

## State Management

**Approach:** Local React state only (`useState`)

**State variables in `App.tsx`:**
- `data: ReportData` â€” the currently active term being edited
- `savedReports: ReportData[]` â€” list of all persisted terms (loaded from localStorage)
- `showPreview: boolean` â€” toggles editor vs. preview mode
- `activeTab: string` â€” which editor section is visible
- `saveStatus: 'idle' | 'saving' | 'success'` â€” save button feedback state
- `isDownloading: boolean` â€” PDF generation in-progress flag

**No context, no reducers, no external state library.**

## Planned Architecture (from CLAUDE.md)

The planned migration transforms this into a full-stack application:

**Backend:** Hono + Node.js (`server/`)
- Entry point: `server/index.ts`
- Auth: `server/auth.ts` â€” Better Auth with Google OAuth + email/password
- Database client: `server/prisma.ts` â€” PrismaClient
- Routes: `server/routes/terms.ts` (CRUD, filtered by `ownerId`), `server/routes/gemini.ts` (proxy)

**Database:** PostgreSQL via Prisma ORM
- Schema: `prisma/schema.prisma`
- `Term` model maps `ReportData` fields; `contratante`, `contratada`, `items`, etc. stored as `Json` columns
- Auth tables generated by `npx better-auth generate`

**Frontend changes:**
- `localStorage` calls replaced by `fetch('/api/terms/...')`
- Gemini calls replaced by `fetch('/api/gemini/improve')` and `fetch('/api/gemini/summary')`
- `src/lib/auth-client.ts` â€” Better Auth React client
- `src/components/AuthGuard.tsx` â€” wraps App to require login
- `services/gemini.ts` deleted (logic moved to backend)

**Deployment:** Docker Compose on port 8095, nginx reverse proxy in front of Hono API

## Error Handling

**Current strategy:** Minimal â€” try/catch only in `services/gemini.ts` and the PDF download fallback.

**Patterns:**
- Gemini errors: caught, original text returned as fallback
- PDF errors: caught, `window.print()` called as fallback
- localStorage parse error: caught and logged to console, silent recovery
- No error boundaries in the React tree

## Cross-Cutting Concerns

**Logging:** `console.error` only, in catch blocks
**Validation:** None â€” all form fields are free-text inputs with no validation
**Authentication:** None in current state; planned via Better Auth
